Instância: - Nome: bitplus-homolog - AMI: Amazon Linux 2023 - Tipo: t2.micro (Free Tier) - Par de chaves: Criar novo (JA CRIADA bitplus.pem) - Security Group: Criar novo (JA CRIADA rds-ec2-6 (sg-0d8914eb4bcecff54)) - SSH (22) - HTTP (80) - HTTPS (443) - Região: Brasil São Paulo (sa-east-1) RDS: - Nome: bitplus - Engine: MySQL - Versão: 8.0.32 - Instância: db.t3.micro - Security Group: Criar novo (JA CRIADA rds-ec2-6 (sg-0d8914eb4bcecff54)) - Região: Brasil São Paulo (sa-east-1) . Criar e configurar chave SSH: ```bash # Criar diretório .ssh com permissões corretas mkdir -p ~/.ssh chmod 700 ~/.ssh # Gerar chave SSH ssh-keygen -t rsa -b 4096 -f ~/.ssh/ec2-bitplus -N "" -C "ec2-bitplus" # Configurar permissões chmod 600 ~/.ssh/ec2-bitplus chmod 644 ~/.ssh/ec2-bitplus.pub # Mostrar a chave pública cat ~/.ssh/ec2-bitplus.pub ``` 2. Adicionar a chave ao GitHub/GitLab: - GitHub: Settings > SSH and GPG keys > New SSH key - GitLab: Preferences > SSH Keys 3. Configurar SSH para usar a chave: ```bash # Criar/editar arquivo de configuração SSH echo "Host github.com IdentityFile ~/.ssh/ec2-bitplus User git" >> ~/.ssh/config chmod 600 ~/.ssh/config ``` 4. Testar conexão SSH: ```bash ssh -T git@github.com ``` ## 5. Instalar Git e Clonar Repositório ```bash # Instalar Git sudo dnf install -y git # Clonar o repositório git clone git@github.com:bitplus-tech/bitbook-api.git ~/bitbook-api ``` ## 6. Configurar Ambiente 1. Tornar o script executável: ```bash chmod +x ~/bitplus/infra/scripts/setup-ec2.sh ``` 2. Executar o script de setup: ```bash ~/bitplus/infra/scripts/setup-ec2.sh ``` /disposicao dos containers - docker compose centralizador em /infra, gerencia: - container bitbook-api - container bittopics-api - container bittrainers-api - container nginx Funcionalidades - Bitbook-api Container principal que executa a aplicação NestJS. Responsável por: Servir a API REST Gerenciar conexões com o banco de dados Processar requisições HTTP Executar validações e regras de negócio Fornecer documentação via Swagger Banco de dados conexao através do RDS - Bittopics-api - bittrainers-api - Nginx descrever funcionalidades, servir as apis para que cada aplicação tenha seu dominio, e etc.... Como executar: - Acessar cd bitplus-backend > cd infra e executar: docker compose up -d --build - Verificar se os containers estão rodando: docker ps
